## 3. Регистры микроконтроллера

### Что такое регистры?

**Регистр** — это ячейка памяти внутри микроконтроллера, через которую мы управляем периферией.

```c
volatile uint8_t button_pressed = 0;
volatile uint8_t button_debounce = 0;
```

**Почему `volatile`?**
- Ключевое слово `volatile` говорит компилятору: «Эта переменная может измениться в любой момент»
- Без `volatile` компилятор может оптимизировать код и не читать переменную каждый раз
- Важно для:
  - Глобальных переменных, изменяемых в прерываниях
  - Регистров периферии

## 4. Инициализация GPIO

### Функция `GPIO_Init()`

```c
void GPIO_Init(void)
{
    RCC->APB2ENR |= RCC_APB2ENR_IOPAEN;
    RCC->APB2ENR |= RCC_APB2ENR_IOPBEN;
    // ...
}
```

### Шаг 1: Включение тактирования

**RCC** (Reset and Clock Control) — блок управления тактированием и сбросом.

```c
RCC->APB2ENR |= RCC_APB2ENR_IOPAEN;
```

**Разбор:**
| Часть | Значение |
|-------|----------|
| `RCC` | Указатель на структуру регистров RCC (базовый адрес 0x40021000) |
| `->` | Оператор доступа к полю структуры через указатель |
| `APB2ENR` | Регистр включения периферии на шине APB2 |
| `\|=` | Побитовое ИЛИ с присваиванием (устанавливаем бит, не трогая остальные) |
| `RCC_APB2ENR_IOPAEN` | Маска битa для включения порта A (бит 2) |

**Аналогия:** Представьте, что периферия — это электроприборы в доме. Прежде чем использовать прибор, нужно включить автомат в щитке. RCC->APB2ENR — это такой «автомат» для периферии.

### Шаг 2: Настройка режима вывода (PB2 — светодиод)

```c
GPIOB->CRL &= ~GPIO_CRL_MODE2;
GPIOB->CRL |= GPIO_CRL_MODE2_1;
GPIOB->CRL &= ~GPIO_CRL_CNF2;
```

**GPIOB** — порт B (базовый адрес 0x40010C00).

**CRL** (Configuration Register Low) — регистр конфигурации для пинов 0-7.

#### Битовая структура регистра CRL:

Каждый пин настраивается 4 битами:
```
Биты 3:0  → Пин 0
Биты 7:4  → Пин 1
Биты 11:8 → Пин 2 (наш PB2)
```

Для каждого пина:
```
MODE[1:0] — Режим (скорость + направление)
CNF[1:0]  — Конфигурация
```

#### Настройка MODE2:

```c
GPIOB->CRL &= ~GPIO_CRL_MODE2;       /* Очистка MODE2[1:0] */
GPIOB->CRL |= GPIO_CRL_MODE2_1;      /* Установка MODE2 = 10 (50 МГц) */
```

| MODE[1:0] | Режим |
|-----------|-------|
| 00 | Вход (reset state) |
| 01 | Выход, 10 МГц |
| 10 | Выход, 50 МГц |
| 11 | Выход, 2 МГц |

#### Настройка CNF2:

```c
GPIOB->CRL &= ~GPIO_CRL_CNF2;        /* CNF2 = 00 — выход push-pull */
```

| CNF[1:0] для выхода | Режим |
|---------------------|-------|
| 00 | Push-pull (推挽) |
| 01 | Open-drain (открытый сток) |
| 10 | Альтернативная функция Push-pull |
| 11 | Альтернативная функция Open-drain |

**Push-pull** — выход может выдавать и 0, и 1.

### Шаг 3: Настройка входа (PA0 — кнопка)

```c
GPIOA->CRL &= ~GPIO_CRL_MODE0;       /* MODE0 = 00 — вход */
GPIOA->CRL &= ~GPIO_CRL_CNF0;        /* Очистка CNF */
GPIOA->CRL |= GPIO_CRL_CNF0_1;       /* CNF0 = 10 — input pull-up/pull-down */
GPIOA->BSRR = GPIO_BSRR_BS0;         /* Включить pull-up */
```

| CNF[1:0] для входа | Режим |
|--------------------|-------|
| 00 | Analog mode |
| 01 | Floating input |
| 10 | Input with pull-up/pull-down |
| 11 | Reserved |

**Pull-up резистор** — внутренний резистор, подтягивающий пин к +3.3V.

**Почему BSRR, а не ODR?**
```c
GPIOA->BSRR = GPIO_BSRR_BS0;  /* Правильно — атомарная операция */
GPIOA->ODR |= GPIO_ODR_ODR0;  /* Тоже работает, но менее безопасно */
```

**BSRR** (Bit Set/Reset Register) — позволяет устанавливать и сбрасывать биты **атомарно** (за одну операцию).

## 5. Настройка прерываний EXTI

### Функция `EXTI_Init()`

```c
void EXTI_Init(void)
{
    RCC->APB2ENR |= RCC_APB2ENR_AFIOEN;
    AFIO->EXTICR[0] &= ~AFIO_EXTICR1_EXTI0;
    EXTI->FTSR |= EXTI_FTSR_FT0;
    EXTI->IMR |= EXTI_IMR_MR0;
    NVIC_EnableIRQ(EXTI0_IRQn);
    NVIC_SetPriority(EXTI0_IRQn, 1);
}
```

### Шаг 1: Включение тактирования AFIO

```c
RCC->APB2ENR |= RCC_APB2ENR_AFIOEN;
```

**AFIO** (Alternate Function I/O) — блок альтернативных функций ввода-вывода.
Нужен для связи GPIO с EXTI.

### Шаг 2: Выбор источника для EXTI0

```c
AFIO->EXTICR[0] &= ~AFIO_EXTICR1_EXTI0;
```

**EXTICR** (External Interrupt Configuration Register) — выбирает, какой порт связать с линией EXTI.

| Значение | Порт |
|----------|------|
| 0000 | PA0 |
| 0001 | PB0 |
| 0010 | PC0 |
| 0011 | PD0 |

Мы оставляем 0 (PA0), поэтому только очищаем биты.

### Шаг 3: Настройка фронта прерывания

```c
EXTI->FTSR |= EXTI_FTSR_FT0;       /* Падающий фронт (1→0) */
EXTI->RTSR &= ~EXTI_RTSR_RT0;      /* Запрет восходящего */
```

**FTSR** (Falling Trigger Selection Register) — выбор падающего фронта.

**Падающий фронт** — переход сигнала с 1 на 0 (когда нажимаем кнопку, подключённую к GND).

```
Сигнал на PA0:
    
    1 ─────────────┐
                   │ нажатие (падающий фронт)
    0 ─────────────┘
                   ↑
              Прерывание!
```

### Шаг 4: Разрешение прерывания

```c
EXTI->IMR |= EXTI_IMR_MR0;
```

**IMR** (Interrupt Mask Register) — разрешает/запрещает прерывания от линий.

### Шаг 5: Настройка NVIC

```c
NVIC_EnableIRQ(EXTI0_IRQn);
NVIC_SetPriority(EXTI0_IRQn, 1);
```

**NVIC** (Nested Vectored Interrupt Controller) — контроллер прерываний Cortex-M3.

| Функция | Назначение |
|---------|------------|
| `NVIC_EnableIRQ()` | Разрешает конкретное прерывание |
| `NVIC_SetPriority()` | Устанавливает приоритет (0 = высший, 15 = низший) |

## 6. Работа с кнопкой и светодиодом

### Чтение кнопки

```c
uint8_t Button_Read(void)
{
    return (GPIOA->IDR & GPIO_IDR_IDR0) ? 1 : 0;
}
```

**IDR** (Input Data Register) — регистр, содержащий текущие уровни на входах.

**Как работает:**
```c
GPIOA->IDR          /* Читает все 16 пинов порта A */
GPIO_IDR_IDR0       /* Маска битa 0 (0x0001) */
&                   /* Побитовое И — проверяем только бит 0 */
```

**Результат:**
- Кнопка отпущена → PA0 = 1 (pull-up) → возвращается 1
- Кнопка нажата → PA0 = 0 (GND) → возвращается 0

### Управление светодиодом

```c
void LED_On(void)  { GPIOB->BSRR = GPIO_BSRR_BS2; }
void LED_Off(void) { GPIOB->BSRR = GPIO_BSRR_BR2; }
void LED_Toggle(void) { GPIOB->ODR ^= GPIO_ODR_ODR2; }
```

| Регистр | Назначение |
|---------|------------|
| BSRR (Bit Set) | Установка бита в 1 (BS2) |
| BSRR (Bit Reset) | Установка бита в 0 (BR2) |
| ODR (Output Data) | Текущее состояние выходов |
| ^= XOR | Инверсия бита |

## 7. Главный цикл

### Функция `main()`

```c
int main(void)
{
    GPIO_Init();      /* 1. Настройка GPIO */
    EXTI_Init();      /* 2. Настройка EXTI */
    __enable_irq();   /* 3. Глобальное разрешение прерываний */
    Delay(100000);    /* 4. Стабилизация */
    
    while (1)
    {
        /* Основной цикл */
    }
}
```

### Последовательность инициализации:

```
┌─────────────────────────────────────────┐
│  1. GPIO_Init()                         │
│     • Включить тактирование RCC         │
│     • Настроить PB2 как выход           │
│     • Настроить PA0 как вход с pull-up  │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  2. EXTI_Init()                         │
│     • Включить AFIO                     │
│     • Связать PA0 с EXTI0               │
│     • Настроить падающий фронт          │
│     • Разрешить прерывание в EXTI       │
│     • Разрешить прерывание в NVIC       │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  3. __enable_irq()                      │
│     • Глобальное разрешение прерываний  │
│     • Специальная инструкция процессора │
└─────────────────────────────────────────┘
```

### Обработчик прерывания (в stm32f1xx_it.c)

```c
void EXTI0_IRQHandler(void)
{
    if (EXTI->PR & EXTI_PR_PR0)  /* Проверка флага */
    {
        if (!button_debounce)    /* Защита от дребезга */
        {
            button_debounce = 1;
            button_pressed = 1;
        }
        
        EXTI->PR = EXTI_PR_PR0;  /* Очистка флага */
    }
}
```

**PR** (Pending Register) — регистр ожидающих прерываний.

**Важно:** Флаг прерывания **обязательно** нужно очистить, иначе прерывание сработает снова немедленно!

## 8. Ключевые концепции

### 1. Побитовые операции

| Операция | Синтаксис | Пример |
|----------|-----------|--------|
| И | `&` | `reg &= ~mask` — очистить биты |
| ИЛИ | `\|` | `reg |= mask` — установить биты |
| Исключающее ИЛИ | `^` | `reg ^= mask` — инвертировать биты |
| НЕ | `~` | `~mask` — инвертировать маску |

### 2. Работа с регистрами

```c
/* Установить биты (не трогая остальные) */
REG |= MASK;

/* Очистить биты (не трогая остальные) */
REG &= ~MASK;

/* Инвертировать биты */
REG ^= MASK;

/* Проверить бит */
if (REG & MASK) { ... }
```

### 3. volatile

```c
volatile uint8_t flag = 0;  /* Может измениться в прерывании */

/* Без volatile компилятор может сделать такую оптимизацию: */
if (flag) {         /* Прочитал один раз */
    if (flag) {     /* Использовал сохранённое значение! */
        // ...
    }
}

/* С volatile — читает каждый раз: */
if (flag) {         /* Прочитал из памяти */
    if (flag) {     /* Прочитал из памяти снова! */
        // ...
    }
}
```

### 4. Карта памяти STM32F103

```
Адресное пространство:

0x0000 0000 ──────────────────┐
                              │ Flash (программа)
0x0800 0000 ──────────────────┤
                              │
0x1FFF FFFF ──────────────────┤
                              │
0x2000 0000 ──────────────────┤ SRAM (переменные)
                              │
0x2000 4FFF ──────────────────┤
                              │
0x4000 0000 ──────────────────┤ Периферия
                              │
0x4001 0000 ──────────────────┤ GPIOA, GPIOB, ...
                              │
0x4002 1000 ──────────────────┤ RCC
                              │
0x4002 3000 ──────────────────┘
```

### 5. Тактирование периферии

Прежде чем использовать любой периферийный блок, нужно включить его тактирование:

```c
/* Шина APB2 (высокоскоростная периферия) */
RCC->APB2ENR |= RCC_APB2ENR_IOPAEN;   /* Порт A */
RCC->APB2ENR |= RCC_APB2ENR_IOPBEN;   /* Порт B */
RCC->APB2ENR |= RCC_APB2ENR_AFIOEN;   /* AFIO */

/* Шина APB1 (низкоскоростная периферия) */
RCC->APB1ENR |= RCC_APB1ENR_TIM2EN;   /* Таймер 2 */
RCC->APB1ENR |= RCC_APB1ENR_USART2EN; /* USART 2 */
```

```c
/* Битовые операции */
```
